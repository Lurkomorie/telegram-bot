def check_safety(user_text: str) -> Tuple[bool, str]:
    """Check if user message contains forbidden content"""
    forbidden_patterns = [
        (["minor", "child", ...], "Content involving minors..."),
        (["incest", "family", ...], "Incestuous content..."),
        (["rape", "force", ...], "Sexual violence..."),
        (["bestiality", ...], "Bestiality..."),
        (["gore", "dismember", ...], "Extreme violent content...")
    ]

    for keywords, reason in forbidden_patterns:
        if any(keyword in text_lower for keyword in keywords):
            return False, reason

    return True, ""

## üîÑ Message Flow Comparison

### Sexsplicit

```
1. User sends message ‚Üí saveMessages()
2. Safety check ‚Üí FLAG or PASS
3. State Resolver (Brain 1) ‚Üí newState
4. Dialogue Specialist (Brain 2) ‚Üí dialogueResponse
5. Save assistant message with state ‚Üí saveMessages()
6. Publish to Ably (real-time)
7. [Background] Image Decider ‚Üí shouldGenerate
8. [Background] Prompt Engineer ‚Üí imagePlan
9. [Background] Runpod dispatch ‚Üí webhook callback
```

### Telegram Bot

```
1. User sends message ‚Üí crud.create_message()
2. Safety check ‚Üí refuse or continue
3. State update ‚Üí update_conversation_state()
4. LLM generation ‚Üí generate_text() with typing action
5. Save assistant message with state ‚Üí crud.create_message()
6. Send message to Telegram
7. [On /image] Build prompts ‚Üí build_image_prompts()
8. [On /image] Create job ‚Üí crud.create_image_job()
9. [On /image] Runpod dispatch ‚Üí webhook callback ‚Üí send_photo()
```

## üîß Configuration Philosophy

### Sexsplicit

- Prompts in database (PromptService)
- Character configs in database
- App settings in code/env

### Telegram Bot

- **All prompts** in `config/prompts.json` (version controlled)
- **All app settings** in `config/app.yaml` (version controlled)
- **Secrets only** in environment variables
- Personas seeded from config on startup

**Advantages:**

- ‚úÖ Easy to version control prompts
- ‚úÖ No database migrations for prompt changes
- ‚úÖ JSON schema validation
- ‚úÖ Can diff prompt changes in Git
- ‚úÖ Simple rollback (just redeploy previous commit)

## üéì Key Learnings

### What's Different (by design)

1. **No streaming**: Telegram doesn't support streaming updates

   - Solution: Typing indicator provides feedback

2. **No Ably**: Telegram has native push

   - Solution: Use Telegram's sendMessage API

3. **Webhook-first**: Telegram requires webhooks in production

   - Solution: FastAPI webhook endpoint

4. **Image async**: Runpod callbacks instead of polling
   - Solution: HMAC-verified webhook + database job tracking

### What's the Same

1. ‚úÖ Pipeline architecture (Safety ‚Üí State ‚Üí Dialogue ‚Üí Image)
2. ‚úÖ Template replacement system
3. ‚úÖ State tracking schema
4. ‚úÖ Retry logic and error handling
5. ‚úÖ Safety filtering
6. ‚úÖ Image prompt engineering
7. ‚úÖ Persona system
8. ‚úÖ Message history management

## üéØ Result

**The Telegram bot is a 1:1 functional replica of Sexsplicit's AI pipeline**, adapted for the Telegram platform while maintaining the same:

- Conversation quality

The implementation proves the pipeline is **platform-agnostic** and can be deployed to any messaging platform (Discord, WhatsApp, Slack, etc.) with minimal changes.
